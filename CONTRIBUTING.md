# Contributing

Thanks for considering a contribution.

## Development

```bash
npm install
npm run check
```

Use `npm run typecheck` when changing TypeScript contracts, package boundaries, or shared runtime behavior.

## Pull Requests

- Keep changes focused.
- Do not commit secrets, real client data, runtime files, generated diagnostics, provider connection ids, or private deployment configuration.
- Add or update E2E coverage for behavior changes when practical.
- Keep public docs generic and avoid private operator runbooks.
